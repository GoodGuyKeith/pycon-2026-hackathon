# Judge Packet

This is the shortest path through the submission. It is meant for reviewers who want to know what to run, what to click, and what the project honestly claims.

## Project

- **Name:** Event Buddy
- **Positioning:** Open Track event companion for PyConSG-style conferences, meetups, and workshops
- **Theme fit:** Architecting more human, useful, and playful event experiences
- **Core idea:** import an agenda, pick goals, get a personalized day plan, verify session attendance with room codes, and count attendee connections with buddy codes.

## Fast Review Path

1. Read `README.md` for run instructions and feature summary.
2. Run `make check`.
3. Run `make run` and open `http://127.0.0.1:8000/index.html`.
4. Follow `DEMO_SCRIPT.md` for the two-minute walkthrough.
5. Run `make smoke` against the live server.
6. Run `make pages-preview-check` and `make final-preflight` before final submission.

## What To Click In The Demo

1. Start with the built-in PyConSG sample.
2. Show the MindMarket-inspired `Human agenda lab` and `Research desk` interface.
3. Paste agenda text or upload CSV / `.ics`.
4. Add one manual `Quick interview card`.
5. Pick an audience template and generate a plan.
6. Enter a visible room code to verify session attendance.
7. Open Buddy, connect with an attendee code, then show Rewards and Venue.

## Criteria Evidence

- **Data Quality:** PyConSG sample agenda, explicit source notes, client-side CSV / `.ics` parsers, and best-effort website import with honest fallback messaging.
- **User Focus:** goals, energy, social mode, venue drift, break protection, introvert-friendly prompts, and attendee connection flow.
- **Overall Experience & Project Value:** warm research-studio visual design, personalized timeline, check-ins, rewards, movement map, copy/save plan.
- **Technical Execution:** React, Tailwind, Vite, Python stdlib backend, localStorage persistence, smoke test, package audit, GitHub Actions, and static preview tooling.

## Honest Boundaries

- Website importing is best effort because event pages vary.
- PDF/image uploads are labeled as OCR-next rather than pretending local OCR exists.
- Saved plans and progress are browser-local.
- The built-in sample is curated from public PyConSG schedule information, not continuously synced.
